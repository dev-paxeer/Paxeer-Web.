Computable Token Machine

Beyond value, beyond utility – tokens that think. Discover Paxeer's groundbreaking CTM standard.

1. Introduction

Beyond Value, Beyond Utility. Tokens That Think.

Traditional tokens are passive assets. The Computable Token Machine (CTM) introduces a third dimension: computation. Each CTM token is a modular execution environment capable of running its own applications, managing state, and evolving functionality over time.

2. Core Concepts

What is a CTM?

A CTM combines a standard token interface with the upgradeable Diamond Standard (EIP-2535), giving it two personas:

  1. The Token: behaves like ERC-20 for transfers & integrations.
  2. The Machine: routes calls to modular Programs (facets) that can be added, replaced, or removed post-deployment.

Key Features

  • Infinite Extensibility – ship new functionality anytime.
  • Shared State – seamless interaction between Programs.
  • Gas Efficiency – bypass contract-size limits via modular code.
  • True On-Chain Autonomy – create complex agents that self-manage assets & logic.

3. Deploy Your First CTM

$ git clone https://github.com/dev-paxeer/Computable-Token-Machine-v1.0.0
$ cd Computable-Token-Machine-v1.0.0
$ npm install
$ npx hardhat run scripts/deploy.js --network <network>

The script deploys the proxy TokenVM.sol and standard facets, returning the permanent CTM address.

4. Creating CTM Programs

Programs are stateless Solidity contracts that live inside the CTM. Example snippet:

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import {ERC20Facet} from "./ERC20Facet.sol";

contract VotingProgram {
  struct VotingStorage { mapping(uint => string) proposals; }
  bytes32 constant SLOT = keccak256("ctm.voting");
  // ...
}
